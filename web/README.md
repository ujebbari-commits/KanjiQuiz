# KanjiQuiz Web

Firefox上でYomitanを使いながら遊べる、KanjiQuizのWeb版です。

## 主な機能

- JSON / CSV / TSV / TXTのデッキ読み込み
- 通常、サバイバル、タイムアタック、3回正解、苦手語、デイリー
- 間違いを全問正解するまで繰り返す定着復習
- 逆4択と類似したダミー選択肢
- 1問ごとの最大挑戦回数
- 履歴、正答率グラフ、苦手度、連続日数
- めくりモード
- オフラインPWA
- 問題文と正解文を通常のHTMLテキストで表示し、Firefox/Yomitanで選択可能

## 対応データ

### KanjiQuiz JSON

```json
{
  "format": "KanjiQuizWebDeck",
  "version": 1,
  "deck": {
    "name": "難読漢字",
    "fields": ["漢字", "読み", "メモ"],
    "notes": [
      {"id": "1", "fields": ["薔薇", "ばら", "植物"]}
    ]
  }
}
```

### CSV / TSV / TXT

先頭行をフィールド名として読み込めます。Anki Desktopのテキスト書き出しはTSVとして読み込めます。AnkiDroidから移す場合は、Android版v1.10のデッキ設定画面で「Web版用JSONを保存」を押してください。

## GitHub Pages

`.github/workflows/pages.yml`が`web/`をGitHub Pagesへ公開します。
リポジトリの Settings → Pages → Source を `GitHub Actions` に設定してください。

公開URLは通常、次の形式です。

`https://<GitHubユーザー名>.github.io/<リポジトリ名>/`


## v0.1.1

Android版Firefoxのファイル選択で `File.text()` が空文字を返す場合に備え、ArrayBufferとFileReaderによる読み込みへ自動的にフォールバックします。選択ファイルが0バイトとして渡された場合は、ファイル管理アプリでDownloadへコピーまたは別名保存する案内を表示します。


## Web v0.1.2

「3回正解」のカード別進捗をlocalStorageへ保存します。正解で+1、不正解で-1となり、3に到達したカードは同じ設定では出題されません。


## Web v0.1.3

ゲーム画面の文字サイズを16〜96pxで設定できます。クイズ中とめくり中も A− / A＋ で変更でき、値はFirefox内に保存されます。


## Web v0.1.5

- 入力欄のフォーカスだけでキーボード表示中と誤判定し、タイマーやパス等が消える問題を修正。
- 実際にViewportが縮んだ場合だけコンパクト表示へ切り替える。
- タイマー、進捗、パス、スコア、文字サイズ操作は常時表示し、問題文領域のみ必要に応じてスクロール可能にした。


## Web v0.1.6

- 再挑戦しても同じ問題の残り時間を引き継ぎます。
- Firefox Androidでキーボード表示中に回答・パスをタップした際、最初のタップで確実に処理されるよう修正しました。

## Web v0.1.7

- 全クイズモードで共有する成功回数設定を追加。
- 成功3回のカードを全クイズモードから除外。
- カード別成功回数画面で検索・確認・カード単位リセットが可能。

## Web v0.1.8

- 全クイズ形式で「新規のみ」を利用可能。カードを問題として表示した時点で新規状態を解除し、Firefox内へ保存します。
- 複数デッキを混ぜる仮想「デイリーデッキ」を追加しました。
- デイリー目標はカード枚数またはプレイ時間から選択できます。
- デイリー対象デッキを複数選択でき、未完了日はホームのデイリーカードが光ります。
- 枚数目標に必要なカードが不足している場合は、目標を自動縮小せずエラーを表示します。
