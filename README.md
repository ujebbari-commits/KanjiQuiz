# KanjiQuiz — 引き継ぎパッケージ

AnkiDroid のデッキを読み取って出題する漢字クイズ Android アプリ。

## このパッケージの中身
- `HANDOFF.md` … **最初に読む引き継ぎ資料**（目的・構成・設計判断・現状・残タスク・注意点）
- `app/`, `build.gradle.kts`, `settings.gradle.kts`, `gradle.properties`, `.github/` … ビルド可能なプロジェクト一式
- 全ロジックは `app/src/main/java/com/example/kanjiquiz/MainActivity.kt`（1ファイル）

## ビルド方法
手元にコンパイラが無くても、GitHub に push すれば `.github/workflows/build.yml` が
debug APK を自動ビルドし、Actions の Artifacts に `KanjiQuiz-v1.17-apk` として出力する。
詳細は `HANDOFF.md` の「3. ビルド／配布パイプライン」を参照。

## 前提（アプリ利用側）
AnkiDroid を一度起動し、設定 → 高度な設定 →「AnkiDroid API」を有効化しておくこと。


## 2026-07-20 追加機能
- サバイバル／タイムアタックの循環出題
- 定着するまで復習
- 苦手語チャレンジ
- 逆モードの類似選択肢
- デイリーチャレンジ
- タイムアタックの時間加算・コンボ演出
- デッキ／モード／出題方向別の履歴フィルター

## v1.1
- 各カードが合計3回正解するまで循環する「3回正解」モード
- クイズ／めくり画面のカード本文を長押しで選択可能（Yomitanなどの選択メニュー用）
- APKメタデータとGitHub Actions成果物名をv1.1に更新

## v1.2
- 1問ごとの最大挑戦回数を設定できる機能を追加（初期値3回、1〜99回）
- 途中の誤答では正解を表示せず、同じカードへ再挑戦
- 全挑戦を使い切った場合だけ、そのカードを不正解として履歴・苦手度・サバイバル残機へ反映
- APKメタデータとGitHub Actions成果物名をv1.2に更新


## v1.4 / Web v0.1.2

- 「3回正解」はデッキ・フィールド・出題方向ごとに端末へ累積保存
- 正解で +1、不正解で -1（0未満にはならない）
- 累積3回に達したカードは以後このモードでは出題しない
- 現在の設定または全設定の累積進捗をリセット可能


## v1.7 / Web v0.1.5

- 入力欄のフォーカスだけでキーボード表示中と誤判定し、タイマーやパス等が消える問題を修正。
- タイマー、進捗、パス、スコア、文字サイズ操作はキーボードの有無にかかわらず常時表示。
- 実際にソフトキーボードが開いた場合だけ余白を縮め、問題文領域を内部スクロール可能にした。

## v1.5 / Web v0.1.3

- ゲーム画面のカード文字サイズを16〜96で設定可能
- クイズ中・めくり中も A− / A＋ で即時変更可能
- 問題、正解表示、逆4択、めくり表裏へ比率を保って反映
- 変更値は端末へ保存し、次回以降も維持


## v1.8 / Web v0.1.6

- 誤答後に同じカードへ再挑戦しても、1問の残り時間をリセットしない。
- Firefox AndroidでIME表示中に回答・パスを押すと、blurによるViewport変更でclickが消える問題を修正。
- Android版にも明示的な「回答」ボタンを追加し、回答・パスの処理後に必要な場合だけIMEを閉じる。

## v1.9 / Web v0.1.7

- 「成功回数を全モードで共通にする」設定を追加。オンの場合、すべてのクイズモードで正解は+1、最終不正解・パス・時間切れは-1（下限0）。
- 共通成功回数が3に達したカードは、通常・サバイバル・タイムアタック・3回正解・苦手語・デイリー・定着復習から除外。
- 共通回数はデッキとカード単位で保存し、フィールド構成・出題方向・ゲーム形式をまたいで共有。
- デッキ設定画面に「カード別の成功回数を見る」を追加。検索、0〜3の確認、カード単位のリセットに対応。
- 共通設定がオフの場合、従来の3回正解モード専用進捗を維持。

## v1.10 / Web v0.1.8

- すべてのクイズ形式で使える「新規のみ」を追加。問題として一度も表示されたことがないカードだけを出題する。
- 新規判定はカードを表示した時点で即保存するため、プレイ途中で終了しても再び新規扱いにはならない。
- 複数の指定デッキを混ぜて、KanjiQuiz内だけの仮想「デイリーデッキ」を生成。
- デイリーの達成条件を「カード枚数」または「プレイ時間」から選択可能。初期値は100枚／30分。
- 未完了日はホームのデイリーカードを強調表示。完了すると当日のボタンを完了表示に切り替える。
- 枚数目標より対象カードが少ない場合は、目標を自動的に下げず、デッキ追加または条件変更を案内する。
- デイリーに含めるデッキは専用設定画面で複数選択可能。各デッキで最後に使った問題側・回答側フィールドを利用する。


## v1.12 / Web v0.1.9

- ホームの「今日のデイリー」下部ボタンを「設定して始める」に変更し、デイリー設定画面を開く。
- 枚数制デイリーは正解したカードだけを達成枚数に加算。不正解・パスしたカードはキュー末尾へ戻し、後で再出題する。
- パス時は正解を表示し、自動送り設定がオンでも「次へ」を押すまで停止する。


## Web v0.1.10

- Firefox Androidでパスまたは回答を短くタップした際に二重処理される問題を修正しました。
- パスは必ず答え表示で停止し、「次へ」を別途押すまでカードを切り替えません。

## Web v0.1.13

Web版で、AnkiDroidから再出力した更新済みデッキを既存デッキへ統合できるようになりました。同じカードIDの進捗を維持し、新規カードだけを進捗0で追加します。


## v1.13 / Web v0.1.14

- デイリー中も、デイリー全体の進捗バーとは別に「現在の問題」の残り時間バーを表示する。
- 枚数制・時間制のどちらでも、各問の制限時間が有効な場合に表示する。
- キーボード表示中も2本のバーとラベルを維持する。

## v1.14 / Web v0.1.15
- デイリー中の「現在の問題」の残り時間バーを、デイリー全体の時間表示とは独立して表示する。
- 時間制デイリーの全体残り時間はデイリー設定で表示・非表示を選択でき、初期値は非表示。
- 全体時間を非表示にしても達成判定と時間計測は継続する。


## v1.15 / Web v0.1.16

- 今日のデイリー完了後も、ホームの「もう一度デイリー」から同じ日に再プレイできる。
- 当日の完了記録とストリークは維持し、再プレイは追加練習として扱う。
- 再プレイの結果は正式なデイリー履歴へ重複登録しない。


## v1.16 / Web v0.1.17

- デイリー中の左上表示を「デイリー・回答 N問」に変更。進捗バーと同じ残り時間・達成枚数は左上に重複表示しない。進捗バー自体は変更なし。


## v1.17 / Web v0.1.19

- めくりモードに端末標準の自動読み上げを追加。
- 読み上げ対象（両面／表面／裏面）、速度、表裏の待ち時間、繰り返し回数、括弧内注釈の有無を設定可能。
- 読み上げONでは表面→裏面→次カードの順に自動進行し、一時停止・前後移動・現在面の再読み上げに対応。
- 読み上げ時だけHTMLタグ、ピッチアクセント矢印などを除外する。
